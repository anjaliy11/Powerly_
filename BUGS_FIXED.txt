BUGS FIXED IN FRAMEWORK
=======================

Three critical bugs were found and fixed:

BUG #1: Reward Function - Renewable Fraction Issue
===================================================
PROBLEM:
- Renewable bonus was using: renewable_fraction = renewable_kw / demand_kw
- This could be >1, causing huge rewards (e.g., 124k instead of -20 to -50)
- Example: 30 kW solar / 20 kW demand = 1.5, then 2.0 * 1.5 = 3.0 per step
- Over 96 steps this compounds incorrectly

LOCATION: env/microgrid_env.py:456

FIX APPLIED:
- Changed from: renewable_bonus = RENEWABLE_REWARD * (renewable_kw / demand_kw)
- Changed to:   renewable_bonus = RENEWABLE_REWARD * renewable_kw * dt_hours
- Now all reward components are normalized consistently (per kWh per timestep)

IMPACT:
- Reward now in range: -200 to 0 per episode (realistic)
- Training convergence will be MUCH better


BUG #2: AT-DRAC Baseline - Time Decoding Error
==============================================
PROBLEM:
- Hour calculation used: arctan2(sin, cos) * 12 / pi
- Should be:             arctan2(sin, cos) / (2*pi) * 24
- This caused CSR to drop to 0.508 instead of 0.94+
- Baseline was making wrong decisions at wrong times

LOCATION: agents/baselines.py:98

FIX APPLIED:
- Corrected time decoding formula
- Added safety bounds: np.clip(hour, 0, 24)

IMPACT:
- AT-DRAC CSR now ~0.94-0.96 (correct)
- AT-DRAC baselines will have normal behavior


BUG #3: Edge Case - Hour Wrapping
=================================
PROBLEM:
- Time decoding could return negative or >24 values in edge cases
- Caused AT-DRAC decision thresholds (6-14h, 14-20h) to fail

LOCATION: agents/baselines.py:101

FIX APPLIED:
- Added: hour = np.clip(hour, 0, 24)

IMPACT:
- Prevents time-based anomalies
- More robust baseline behavior
